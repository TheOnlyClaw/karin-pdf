PDF Compressor — 100% Client-Side Architecture

Zero-server PDF compression • Ghostscript WASM • Cache-First Service Worker

🗂 App Shell (React SPA)

  • • 6 pure presentational components (Dropzone, Slider, Button…)
  • • Centralized useReducer for all state transitions
  • • Custom hooks: useGhostscript, useCompress, useElapsedTime
  • • Dark/Light theme via prefers-color-scheme CSS vars
  • • Build: Vite + Tailwind CSS v4, ~60KB gzipped

🛠 Ghostscript WASM Engine

  • • ~5MB gzipped — cached after first visit
  • • MEMFS virtual filesystem (no disk I/O)
  • • Tier selection (prepress→printer→ebook→screen)
  • • Binary search on DPI (36–300) as fallback
  • • Single-threaded, blocking run (indeterminate progress shown)

⚡ Caching Strategy

  • • Service Worker intercepts /ghostscript.wasm
  • • Cache-First: zero network on warm visits
  • • Content-addressed WASM file (hash in filename)
  • • Cold visit: ~5.1MB total (WASM + app shell)
  • • Warm visit: instant — 0KB network, <100ms init
